# 역할별 진입점 및 유저 플로우 점검

## 0. 프로필 규칙

- **유저 1명당 각 프로필 1개씩 필수**: user_profile(관람객), performer_profile(공연자), host_profile(호스트)를 모두 1개씩 보유.
- 로그인 후 `/profiles/select`에서 활성 모드(ROLE_USER / ROLE_PERFORMER / ROLE_HOST)를 전환하여 해당 역할로 이용.

## 1. 역할 구분

| 역할 | activeMode | 용도 |
|------|------------|------|
| **게스트** | (비로그인) | 공연/공연장 탐색, 피드, 로그인 |
| **유저(관람객)** | ROLE_USER | 공연 보기·리뷰, 공연장 상세·좌석 리뷰 |
| **공연자** | ROLE_PERFORMER | 모집글, 공연장 대관 신청, 내 대관/모집 관리 |
| **호스트** | ROLE_HOST | 내 공연장, 대관 요청 관리 |

---

## 2. 푸터 진입점 (현재)

| 역할 | 메뉴 | 이동 경로 |
|------|------|-----------|
| **게스트** | 공연리스트, 공연장, 피드, 로그인 | /performances, /venues, /feed, /auth/login |
| **유저** | 공연리스트, 공연장, 피드, DM, 채팅 | /performances, /venues, /feed, /dm/list, /chats |
| **공연자** | 모집글, 공연장, 홈, DM, 채팅 | /posts/performance, /venues, /, /dm/list, /chats |
| **호스트** | 내 공연장, 대관관리, 홈, DM, 채팅 | /venues/my, /venues/reservations/manage, /, /dm/list, /chats |

## 3. 역할별 유저 플로우 요약

### 3.1 게스트
- **진입**: 공연리스트 / 공연장 / 피드 / 로그인
- **플로우**: 공연·공연장 목록 조회, 피드 탐색 → 로그인 시 `/auth/login` → (로그인 성공 후 프로필 선택 등)
- **평가**: 적절함.

### 3.2 유저(관람객)
- **진입**: 공연리스트, 공연장, 피드, DM, 채팅
- **주요 플로우**
  - **공연**: `/performances` → 상세 `/performances/{id}` → (본공연 탭) 공연 리뷰 `/performances/{id}/reviews/new`, 좌석 리뷰 `/performances/{id}/reviews/seats/new`
  - **공연장**: `/venues` → 상세 `/venues/{id}` → 공연 선택 후 좌석 리뷰 작성 → `/performances/{pid}/reviews/seats/new`
- **리뷰**: 관람객(ROLE_USER) 전용, API에서 activeMode 검증
- **이슈**
  - **마이페이지 진입점 없음**: 푸터에 "마이페이지" 링크가 없어 `/mypage` 접근이 URL 직접 입력에 의존함. (헤더 프로필 아이콘은 `targetUrl`로 현재 페이지로 이동)

### 3.3 공연자
- **진입**: 모집글, 공연장, 홈, DM, 채팅
- **주요 플로우**
  - **모집글**: `/posts/performance` 목록·작성·상세·수정
  - **대관 신청**: `/venues` → 카드 클릭 → `/venues/{id}/reservation` (대관 폼)
  - **내 대관 신청 목록**: `/venues/reservations/my` (대기/승인/거절 확인)
- **마이페이지(공연자 설정)**: `/mypage` → 내 모집글, 신청한 글, 모집한 공연자, **내가 대관한 공연장** (`/mypage/performer/venues`), 신청자 관리
- **이슈**
  - **"내 대관 신청 목록" 진입점 부족**: 이전에는 `/venues/performer`(performerVenueList) 상단에 "내 대관 신청" 버튼으로 `/venues/reservations/my` 연결이 있었음. 목록을 `/venues` 하나로 통합한 뒤 해당 버튼이 사라져, 공연자가 "내 대관 신청 목록"에 가는 경로가 푸터/마이페이지에 없음.

### 3.4 호스트
- **진입**: 내 공연장, 대관관리, 홈, DM, 채팅
- **주요 플로우**
  - **내 공연장**: `/venues/my` → 등록·수정·대관 관리 진입
  - **대관 요청 관리**: `/venues/reservations/manage` → 요청 목록·승인/거절
- **마이페이지(호스트 설정)**: `/mypage` → 호스트 프로필 보기/설정, 내 공연장 관리, 대관 요청 관리 (푸터와 중복이지만 진입용으로 적절)
- **평가**: 적절함.

## 4. 개선 제안

### 4.1 공연자 — "내 대관 신청 목록" 진입점 추가 (권장)
- **위치**: 마이페이지 > 공연자 설정
- **내용**: "내 대관 신청 목록" 메뉴 추가 → `/venues/reservations/my` 링크
- **이유**: 대관 신청 후 상태 확인(대기/승인/거절) 경로를 공연자에게 명확히 제공

### 4.2 유저/공연자/호스트 — 마이페이지 진입 (선택)
- **현재**: 푸터에 "마이페이지" 없음. 헤더 프로필은 `targetUrl`(현재 페이지)로만 이동.
- **선택지**:
  - 푸터에 "마이페이지" 추가 (유저·공연자·호스트 공통 또는 역할별),
  - 또는 헤더 프로필 클릭 시 `/mypage`로 통일 (현재는 페이지별로 상이)
- **비고**: 로그인 후 리다이렉트나 프로필 선택 화면에서 `/mypage`로 보내는 방식이 있다면, 푸터 추가는 선택 사항.

### 4.3 기타
- **공연장 목록**: `/venues` 단일 목록, 프로필별 카드 클릭 분기(유저 → 상세, 공연자 → 대관 신청) 유지하면 역할 분리 적절함.
- **채팅/DM**: 실시간·소켓 담당자 별도로 두기로 한 부분 반영됨.
