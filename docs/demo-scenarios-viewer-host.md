## 데모 시나리오 (필수) — 관람객 / 호스트 기준

> 발표 25분 중 **실제 시연에 꼭 넣을 최소 시나리오**만 정리했습니다.  
> 역할별 흐름이 자연스럽게 이어지도록, 관람객 → 호스트 순으로 구성합니다.

---

## 1. 관람객 프로필 기준 필수 시연

### 1-1. 관람객으로 로그인 & 프로필 확인

- **목표**: “관람객으로 이 플랫폼을 사용한다”는 출발점 보여주기.
- **경로**
  - (필요 시) `/auth/login` → 관람객 계정 로그인
  - 상단 프로필 드롭다운 → `마이페이지(/mypage)`
- **포인트**
  - `관람객 프로필` 섹션에서:
    - 프로필 보기/설정 메뉴 노출
  - `관람객 활동` 섹션:
    - `내가 본 공연`, `내가 찜한 공연` 메뉴가 있다는 것만 짧게 언급
  - “지금부터는 관람객 시점으로 이동하면서, 나중에 다시 이 마이페이지로 돌아올 수 있다” 정도만 설명

---

### 1-2. 피드에서 공연 발견 → 공연 상세 진입

- **목표**: 피드를 통해 공연/사람들이 보이는 느낌 전달.
- **경로**
  - `/feed`
  - 피드 카드 중 하나 클릭 → `/performances/{id}`
- **포인트**
  - 피드에서 최소 1~2개 카드만 강조:
    - “다가오는 공연” 또는 “OOO님이 찜한 공연”
  - 클릭 후 공연 상세로 자연스럽게 이어지는 흐름만 보여주면 충분

---

### 1-3. 공연 상세에서 정보 + 본공연 + 채팅 CTA

- **목표**: 관람 경험 → 채팅으로 자연스럽게 이어지는 UX.
- **경로**
  - `/performances/{id}`
- **포인트**
  - 상단:
    - 포스터, 카테고리, 찜/본공연 버튼
  - 중간:
    - `지역 · 공연장이름` (클릭 가능)  
    - “이 공연의 아티스트” / “이 공연이 열리는 무대” 라벨
  - **행동**
    - `본 공연` 버튼 클릭
    - → 지도 아래에 뜨는 배너:
      - “이 공연을 본 관객들과 이야기 나눠볼까요?”
      - `공연 채팅 참여` 버튼 클릭 → `/performances/{id}/chats`
  - 채팅 화면에서는 “공연을 본 관객들이 모여있는 공간” 정도만 간단히 보여주고 끝

---

### 1-4. 공연장 상세에서 “이 공연장에서의 경험들”

- **목표**: 공연장 단위로도 경험이 모여 있다는 점 보여주기.
- **경로**
  - 공연 상세 상단의 `지역 · 공연장이름` 클릭 → `/venues/{venueId}`
- **포인트**
  - 상단 카드:
    - 공연장 이름, 주소, 연락처, 이미지, 좌석 수
  - “이 공연장에서의 경험들” 섹션:
    - `이 공연장에서 열리는 공연` 리스트 (있다면 2~3개만 스크롤)
    - `좌석 리뷰` 안내 문구 + 공연 선택 드롭다운 정도만 가볍게 언급
  - 다시 공연 상세로 돌아갈 수 있다는 정도만 짚고 관람객 시나리오 종료

---

## 2. 호스트 프로필 기준 필수 시연

### 2-1. 호스트로 로그인 & 내 공연장 확인

- **목표**: “공연장을 가진 사람” 시점에서 이 플랫폼을 보게 하기.
- **경로**
  - (필요 시) `/auth/login` → 호스트 계정 로그인
  - 상단 프로필 → `마이페이지(/mypage)` 진입
- **포인트**
  - 마이페이지에서 `호스트 설정` 섹션:
    - `호스트 프로필 보기/설정`
    - `내 공연장 관리`, `대관 요청 관리` 메뉴 존재만 짧게 보여주기

---

### 2-2. 내 공연장 목록 → 공연장 상세

- **목표**: 호스트가 만든 공연장이, 관람객 플로우의 출발점이 된다는 것 설명.
- **경로**
  - 마이페이지(호스트 모드) → `내 공연장 관리` → (`/venues/my` 혹은 대응 화면)
  - 리스트에서 공연장 하나 클릭 → `/venues/{venueId}`
- **포인트**
  - “이 공연장은 호스트가 등록/관리하는 공간”이라는 한 줄 설명
  - 이어서 관람객 시나리오에서 봤던 것과 같은:
    - 상단 공연장 카드
    - “이 공연장에서의 경험들” 섹션  
    를 다시 한 번 짧게 짚어주면서,
  - “호스트가 만든 공연장이, 관람객과 공연자를 잇는 무대”라는 메시지를 강조

---

### 2-3. (옵션, 시간 여유 있을 때만) 대관/신청 흐름 한 번 언급

- **목표**: 상세 시연까지는 아니고, 기능 존재만 알려주기.
- **경로**
  - 호스트 마이페이지 → `대관 요청 관리`
  - 공연자 마이페이지(또는 공연장 상세)에서 `대관 신청` 버튼이 있다는 정도만 슬라이드로 보여주기
- **포인트**
  - 실제 API 호출/상태 변경까지는 생략해도 됨
  - “공연장–공연자–관람객” 세 축이 모두 연결된다는 구조만 말로 설명

---

## 요약: 25분 발표에서 꼭 가져갈 흐름

1. **관람객 프로필**  
   - 피드 → 공연 상세(본공연 + 채팅 CTA) → 공연장 상세(이 공연장에서의 경험들)
2. **호스트 프로필**  
   - 마이페이지(호스트) → 내 공연장 관리 → 공연장 상세  
   - “이 무대를 통해 관람객/공연자가 이어진다”는 메시지로 관람객 시나리오와 연결
