## 남은 일 정리 (Encore)

> 기능 개발은 큰 줄기 기준으로 마무리된 상태이고,  
> 지금부터는 **데이터 세팅 · QA · 디자인/문서 다듬기** 단계입니다.

---

## 1. 시연용 데이터 세팅

- **계정/프로필**
  - [ ] 관람객 계정 1개: 관람객 프로필 설정 완료.
  - [ ] 공연자 계정 1개: 공연자 프로필 설정 + 공연 1~2개 등록.
  - [ ] 호스트 계정 1개: 호스트 프로필 설정 + 공연장 1~2개 등록.

- **공연장/좌석**
  - [ ] 공연장 최소 3개 (타입/규모 다르게).
  - [ ] 각 공연장에 층/좌석 정보 20석 내외 (좌표/seatNumber/seatType 세팅).

- **공연/스케줄/관람 상태**
  - [ ] 공연 최소 5개 (서로 다른 공연장/장르).
  - [ ] 주요 공연에 PerformanceSchedule 2개 이상 (곧 시작 / 일반).
  - [ ] 관람객 기준:
    - [ ] WATCHED(본 공연) 3~4건.
    - [ ] WISHED(찜 공연) 3~4건 (곧 시작 스케줄 포함).

- **리뷰/좌석리뷰/피드용**
  - [ ] 공연 후기(Review seat=null) 공연별 3~5개 (관람객 본인 후기 1개 이상).
  - [ ] 좌석 리뷰(Review seat!=null) 좌석별 1개 이상 (5~10좌석).
  - [ ] 팔로우/차단 관계:
    - [ ] 관람객 → 공연자 팔로우 1~2건.
    - [ ] 관람객 → 관람객 팔로우 1건.
    - [ ] 차단 1~2건.

## 2. 필수 시연 시나리오 리허설

### 2-1. 관람객 플로우

- [ ] `/mypage` (관람객):  
  관람객 프로필/내 활동/공통 설정 메뉴 구조 확인.
- [ ] `/feed`:  
  다가오는 찜 공연, 친구 추천, 최근 후기/좌석리뷰, 리마인드 카드 확인.
- [ ] `/performances`:  
  검색, 카테고리 탭, 핫 캐러셀, 더보기 버튼 동작 확인.
- [ ] `/performances/{id}`:
  - [ ] 찜/본공연/신고 버튼 동작.
  - [ ] “이 공연의 아티스트/무대” 블록.
  - [ ] 본공연 클릭 시 채팅 CTA 배너 노출 + `/performances/{id}/chats` 이동.
  - [ ] 공연후기 탭: 정렬/더보기/리뷰 작성/상세 모달.
  - [ ] 좌석 리뷰 탭: 좌석 배치도 · 툴팁 · 좌석 리뷰 작성 링크.
- [ ] `/venues/{venueId}`:
  - [ ] 공연장 정보 카드(이미지/주소/연락처/운영시간/좌석 수/시설/휴무).
  - [ ] “이 공연장에서의 경험들” 섹션:
    - [ ] 이 공연장에서 열리는 공연 리스트.
    - [ ] 좌석 리뷰 작성 섹션(관람객 모드일 때만).

### 2-2. 호스트 플로우

- [ ] `/mypage` (호스트):  
  호스트 프로필/내 공연장/대관 요청 관리 메뉴 확인.
- [ ] `/venues/my`:  
  내 공연장 목록 진입 → `/venues/{venueId}`로 연결.
- [ ] 공연장 상세가 관람객 시나리오와 동일 화면이라는 점 설명.

### 2-3. 공연자 플로우 (필수 최소)

- [ ] `/performances` (공연자 모드):  
  핫 캐러셀 우측 “공연등록” 버튼 노출 확인.
- [ ] `/performances/new`:
  - [ ] 공연 등록 폼 입력/저장 → `/performances/{id}` 리다이렉트.
- [ ] `/performances/{id}` (creator로 접속):
  - [ ] 하단 “공연 수정/삭제” 버튼 노출(본인 공연일 때만).
  - [ ] “공연 수정” → `/performances/{id}/edit` → 수정 → 저장 → 상세 복귀.
  - [ ] “공연 삭제” → 리스트로 이동/숨김 확인.

## 3. CSS/레이아웃 정리 (구조 기준)

- [ ] `fragments/profile.html`에서 CSS/JS `<link>/<script>` 제거 (완료).
- [ ] `fragments/topbar.html`에서만 `topbar.css` / `profile.css` / `profile.js` 로딩.
- [ ] 각 페이지의 `<main>`에 역할별 class 부여:
  - [ ] 공연 리스트: `class="app-main performance-list-main"`.
  - [ ] 공연 상세: `class="app-main performance-detail-main"`.
  - [ ] 공연장 상세: `class="app-main venue-detail-main"`.
- [ ] 공연 등록 화면:
  - [x] `performance/new.html` 내장 `<style>` → `css/performance/new.css`로 분리.
  - [ ] 레이아웃/타이포를 `detail.css` vs `new.css`로 적절히 분리/정리.
- [ ] 이후 여유 있을 때:
  - [ ] `community/mypage/performer/*.html` 등 내장 `<style>` 블록을 전용 CSS로 추출.
