# 명세·요구사항 대비 구현 점검

기능 명세서, 사용자 요구사항 정의서, 인터랙션 매트릭스 기준 정리.  
데이터 입력 SQL은 **수정 반영된 상태**이며, 알림 테스트용 `performance_schedule` INSERT 추가됨.

---

## 1. 데이터 SQL (data-test-insert.sql)

| 항목 | 상태 |
|------|------|
| performance `category`, `status` 컬럼 반영 | ✅ 반영 (MUSICAL/PLAY/BAND, UPCOMING/ENDED 등) |
| recruit_status = 모집 상태, category = 장르, status = 진행 상태 | ✅ 주석·값 일치 |
| performance_schedule | ✅ 추가 (공연 1,2,3 일정; 공연 2 = 25분 후 → 북마크 알림 테스트용) |
| users, user_profile, performer_profile, host_profile | ✅ |
| venue, seat, performance, user_performance_relation | ✅ |
| review(공연/좌석 리뷰), reservation, post 등 | ✅ |
| 채팅방/DM 테스트 데이터 | ❌ 없음 (필요 시 별도 보강) |

---

## 2. 관람객(User) 요구사항

| ID | 요구사항 | 구현 |
|----|----------|------|
| U-1 | 공연 리뷰 / 좌석별 리뷰 분리 | ✅ 공연 리뷰·좌석 리뷰 엔티티·API 분리 |
| U-2 | 공연 북마크 + 마이페이지 확인 | ✅ WISHED 관계, 마이페이지 "찜한 공연" 진입 |
| U-3 | 채팅방 생성·참가 (공연 전/후, 태그) | ✅ 생성·참가 가능. **채팅 유형 태그**(같이 갈 사람/애프터/택시/리뷰) 미반영 |
| U-4 | 공연 시작 전 알림 | ✅ 찜한 공연 기준 30분 전 피드 노출 (performance_schedule 연동) |

---

## 3. 공연자(Artist) 요구사항

| ID | 요구사항 | 구현 |
|----|----------|------|
| A-1 | 파티 매칭 (파트·1분 음원) | ⚠️ 모집글(post) 수준. **파트 기준 매칭·음원 필드** 없음 |
| A-2 | 공연장 대관 신청·승인/거절 | ✅ 대관 신청·승인/거절 구현 |
| A-3 | 공연장 리뷰 등록/조회 | ⏸️ **미도입 예정** (공연/좌석 리뷰만 유지) |
| A-4 | 공연 등록 (세트리스트 필수, 공연장 미확정 가능) | ⚠️ 공연장 미확정(venue nullable) 가능. **세트리스트** 필드/테이블 없음 |
| A-5 | 긴급 매칭 (노쇼 대체) | ❌ **미구현** (긴급/emergency/노쇼 관련 코드 없음) |

---

## 4. 공연장 제공자(Venue Provider) 요구사항

| ID | 요구사항 | 구현 |
|----|----------|------|
| V-1 | 공연장 등록 (위치, 접근, 좌석 구조, 화장실 변기 수) | ⚠️ 위치·설명·좌석 수 등 있음. **화장실/변기 수** 전용 필드 없음 |
| V-2 | 공연 등록·공연자 참여 신청 | ✅ 공연 등록·참여 신청 구조 있음 |
| V-3 | 공연자 모집·제안/승인 | ✅ post·interaction 등으로 부분 구현 |

---

## 5. 관리자(Admin) 요구사항

| ID | 요구사항 | 구현 |
|----|----------|------|
| M-1 | 회원 관리·블랙회원 지정 | ⏸️ **추후 개발** (Report·신고 구조만 존재) |
| M-2 | 공연/공연장 관리·비공개 처리 | ⏸️ **추후 개발** |
| M-3 | 게시판·채팅방 관리 | ⏸️ **추후 개발** |
| M-4 | 리뷰 관리·신고 검토·삭제 | ⏸️ **추후 개발** |

---

## 6. 인터랙션 매트릭스 요약

- **관람객/공연자/호스트**: 조회·팔로우·차단·신고·리뷰·북마크·채팅·DM 대부분 대응.
- **미반영/부분**: 공연장 리뷰, 긴급 DM/긴급 모집, 세트리스트, 화장실 정보, 관리자 전용 화면.

---

## 7. 사용자 메모·미확정 사항

| 항목 | 상태 |
|------|------|
| 조회 화면 합치기 (조회 페이지 모아두기) | 미적용 |
| DM UI·긴급 DM 로직·채팅방 UI 확정 | 미확정 |
| 좌석 | **호스트가 공연장 등록 시 직접 입력** (`venueForm.js` STEP 2: 층 추가, 캔버스에 좌석 추가/이동/삭제, 등급 VIP·R·S·A, 좌석번호 입력, 최소 1석 필수) |

---

## 8. 권장 우선순위 (추가 구현 시)

- **미도입·추후**: 공연장 리뷰(A-3), 관리자(M-1~M-4) → 제외 또는 추후 개발로 보류.
- **좌석**: 호스트가 공연장 등록 폼(`venueForm.js` + `venueForm.html`)에서 직접 입력 완료.
- 남은 검토 후보: **세트리스트**(A-4), **긴급 매칭**(A-5), **채팅 유형 태그**(U-3), **화장실/변기 수**(V-1).
