## 기능 접근 권한 매트릭스 (비로그인 / 관람객 / 공연자 / 호스트)

> **범위**: 주요 화면·메뉴·행동 위주로 정리했습니다.  
> 실제 Spring Security 설정과 100% 동일하다고 보장하긴 어렵지만, 현재 템플릿·컨트롤러 기준으로 역할별 사용 가능 범위를 맞췄습니다.

표 기호:
- **O**: 기본 허용
- **X**: 접근 불가 (로그인/역할 필요)
- **조건**: 추가 조건(본인 소유 여부 등)에 따라 달라짐

## 공연장/공연 리스트 → 공연장 상세 UI 개선 제안

요청 주신 “실제 보여지는 정보/UX” 기준으로만 정리했습니다.

### A. 공연장 리스트 → 공연장 상세

- **리스트 카드에 들어가면 좋은 정보**  
  - 공연장 이름, 간단 주소(동/구 단위), 대표 사진/썸네일  
  - 평균 좌석 리뷰 평점 + 리뷰 수 (있다면)  
  - 수용 인원 대략 범위, 주 사용 장르/공연 형태(밴드/연극/뮤지컬 등) 태그
- **상세 상단(공연장 정보 카드)에서 보강하면 좋은 요소**  
  - 리스트에서 본 정보 + 더 자세한 주소, 연락처, 수용 인원, 층/좌석 구조 미리보기  
  - “이 공연장에서 열리는 예정 공연” 섹션: 3~5개 카드로 미리 노출  
  - 좌석 리뷰 요약 박스: 평점, 최근 1~2개의 키워드(“시야 좋음”, “음향 좋음” 등)
- **“어디서 왔는지”에 따른 UX**  
  - 리스트에서 클릭해 온 경우: 상단에 작은 브레드크럼/텍스트 “공연장 리스트로 돌아가기” 링크 유지  
  - 다른 곳(공연 상세 등)에서 온 경우: “이 공연장의 다른 공연 보기” CTA를 좀 더 강조

### B. 공연 리스트/공연 상세 → 공연장 상세

- **공연 상세 내 공연장 정보 표현 개선**  
  - 현재 `지역 · 공연장이름` 텍스트를 **클릭 가능한 링크/버튼**으로 만들어 `/venues/{id}`로 이동 가능하게 하기  
  - 옆의 `지도` 버튼은 지도(카카오/네이버)로 연결, 공연장 상세는 별도의 텍스트 링크 또는 “공연장 상세보기” 버튼으로 분리
- **공연장 상세에서 “이 공연과의 연결” 강화**  
  - 공연 상세에서 넘어온 경우, 공연장 상세 상단/중단에 “지금 보고 있던 공연” 카드(제목/날짜/포스터 축소)를 노출  
  - 그 아래에 “이 공연장 다른 공연” 리스트를 붙여, **현재 공연과 같은/비슷한 장르의 다른 공연**을 추천
- **역할별 디테일**  
  - 관람객(ROLE_USER): “내 위치 기준 거리” 또는 “내 선호 장르와 얼마나 잘 맞는지” 뱃지 노출 (유저 프로필 기반)  
  - 공연자(ROLE_PERFORMER): 같은 공연장에 걸려 있는 **빈 슬롯/대관 가능일**을 한눈에 보여주는 작은 캘린더 또는 “대관 가능 날짜 보기” 버튼  
  - 호스트(ROLE_HOST): 자신의 공연장인 경우, 상세 상단에 “공연장 관리로 가기” 버튼을 배치해 관리 화면으로 바로 이동

이 문서를 베이스로, 추가로 빠져 있는 기능(예: 특정 커뮤니티 화면, 세부 채팅 기능 등)이 있으면 항목만 더 추가하면 같은 형식으로 계속 확장할 수 있습니다.
