# 단일 계정으로 전체 기능 테스트

한 계정만 로그인한 상태로 **관람객 / 공연자 / 호스트** 기능을 모두 테스트할 수 있습니다.

---

## 테스트용 계정

| 항목 | 값 |
|------|-----|
| **이메일** | `user1@test.com` |
| **비밀번호** | `password123` |

- `data-test-insert.sql` 적용 시 **user_id=1** 에 관람객·공연자·호스트 프로필이 모두 들어가 있으며, **초기화 완료** 상태입니다.
- 로그인 후 **프로필 변경**(`/profiles/select`)에서 역할만 바꿔 가며 사용하면 됩니다.

---

## 테스트 순서 예시

1. **로그인**  
   `/auth/login` → `user1@test.com` / `password123`

2. **프로필 선택**  
   첫 로그인 시 `/profiles/select`로 이동 → **관람객** 선택 (또는 이미 선택된 역할 유지)

3. **관람객 기능**  
   - 마이페이지 → 관람객 프로필 보기/설정, 내가 본 공연, 내가 찜한 공연  
   - 공연 목록/상세 → 찜·본공연 토글, 공연 후기/좌석 리뷰  
   - 공연장 목록/상세 → 좌석 리뷰 열람  
   - 피드, 팔로우/팔로워, 차단 목록, DM, 그룹 채팅 등

4. **공연자로 전환**  
   - 상단 프로필 또는 마이페이지 → **프로필 변경** → **공연자** 선택

5. **공연자 기능**  
   - 공연 등록/수정/삭제, 공연자 모집글·공연 모집글 CRUD  
   - 대관 신청, 신청자 관리, 내 대관 신청 목록 등

6. **호스트로 전환**  
   - **프로필 변경** → **주최자** 선택

7. **호스트 기능**  
   - 내 공연장 관리, 대관 요청 승인/거절  
   - 호스트 프로필 보기/설정 등

---

## 요약

- **한 계정**(`user1@test.com`)으로 로그인한 뒤  
- **프로필 변경**으로 관람객 ↔ 공연자 ↔ 호스트만 바꿔 주면  
- **모든 역할의 기능을 한 번에 테스트**할 수 있습니다.
